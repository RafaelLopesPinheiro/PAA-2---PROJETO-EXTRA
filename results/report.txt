2. DESCRIÇÃO DO PROBLEMA
================================================================================

O Vehicle Routing Problem with Time Windows (VRPTW) é um problema
clássico de otimização combinatória classificado como NP-difícil.

Objetivo:
  Minimizar a distância total percorrida por uma frota de veículos
  para atender todos os clientes, respeitando:
    • Capacidade dos veículos
    • Janelas de tempo de cada cliente
    • Tempo máximo de rota

================================================================================
3. ALGORITMOS IMPLEMENTADOS
================================================================================

3.1. Heurística de Inserção de Solomon (I1)
--------------------------------------------------------------------------------
Heurística construtiva gulosa que:
  1. Seleciona cliente inicial (mais distante do depot)
  2. Insere clientes usando critério de custo c(i,u,j):
     c(i,u,j) = α·c1(i,u,j) + λ·c2(i,u,j)
     onde:
       c1 = custo de distância adicional
       c2 = custo temporal (urgência)
  3. Repete até não ser possível inserir mais clientes

3.2. Algoritmo Genético Híbrido
--------------------------------------------------------------------------------
Componentes principais:

a) Representação:
   • Cromossomo = sequência de clientes agrupados em rotas

b) Inicialização:
   • 30% - Heurística de Solomon com parâmetros variados
   • 40% - Construção aleatória gulosa
   • 30% - Mutações da melhor solução

c) Operadores Genéticos:
   • Seleção: Torneio (tamanho 5)
   • Crossover: Order Crossover (OX)
   • Mutação: Swap, Insertion, Inversion

d) Busca Local:
   • 2-opt intra-rota

e) Estratégias Avançadas:
   • Elitismo
   • Reinicialização adaptativa (estagnação > 50 gerações)

================================================================================
4. ANÁLISE DE COMPLEXIDADE
================================================================================

4.1. Heurística de Solomon
--------------------------------------------------------------------------------
Complexidade de Tempo: O(n³)
  onde n = número de clientes

Justificativa:
  • Para cada veículo: O(n)
  • Para cada cliente não roteado: O(n)
  • Teste de inserção em cada posição: O(n)
  • Total: O(n) × O(n) × O(n) = O(n³)

4.2. Algoritmo Genético
--------------------------------------------------------------------------------
Complexidade de Tempo: O(G × P × n²)
  onde:
    G = número de gerações
    P = tamanho da população
    n = número de clientes

Justificativa:
  • Avaliação de fitness: O(n) por solução
  • Crossover (OX): O(n) por operação
  • Mutação: O(1) por operação
  • Busca local 2-opt: O(n²) por solução
  • Por geração: P × O(n²)
  • Total: G × P × O(n²)
